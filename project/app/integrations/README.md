Основные положения по интеграциям:

эндпоинты которые должны осуществляться интеграцией
1) get_salings 
    Получить список продаж
    При получении списка использовать аккаунты парсеры
    в других эндпоинтах по айди аккаунта

2) get_balance
    получить баланс на маркете

3) buy_nft
    Купить нфт (какой либо из списка продаж) на маркетплейсе

4) sell_nft
    Выставить нфт на продажу (из хранилища)

5) cancel_nft
    Снять нфт с продажи

6) back_nft
    Вывести нфт на аккаунт из макретплейса (из хранилища)

7) add_nft
    Внести нфт с аккаунта на маркетплейс (из аккаунта)

8) get_nfts
    Получить список своих нфт на маркетплейсе (само хранилище)

9) get_deals
    Получить сделки выполненные на маркетплейсе

10) get_offers
    Получить список офферов на маркетплейсе

11) accept_offer
    Принять оффер (из списка)

12) cancel_offer
    Отклонить оффер (из списка)

13) new_offer
    Создать оффер


14) get_floors
15) get_floor
    Эти две функции подтягивают данные из базы
    Вытекающие функции:
        1) run_floors_parsing():
            Парсит минимальные цены моделей и коллекций раз в час
            Связанные модели:
                market.MarketFloor - хранит флор по коллекции и модели
